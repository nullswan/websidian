# Tags View

**Date**: 2026-04-04
**Status**: Complete

## What was built

### Server endpoint
- `GET /api/vault/tags` — scans all notes for tags (frontmatter + inline), deduplicates paths per tag
- Returns: `{ tags: [{ name, count, paths }] }` sorted alphabetically

### Tags component
- Right sidebar panel between Outline and CSS Snippets
- Header: "TAGS (N)" with total count
- Nested tag hierarchy: `nested/tag` renders as `#nested` → `#tag` with indentation
- Click a tag to expand and see note paths
- Click a note path to navigate to it
- Tags shown in purple accent color with counts right-aligned
- Collapse/expand with ▶/▼ arrows

### Tag tree builder
- `buildTagTree()` splits tag names by `/` and builds a tree structure
- Each node: { name, fullName, count, paths, children }
