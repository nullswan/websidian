# Obsidian Web Skill

Use this skill when working in this repository.

## Goal

Move the project forward autonomously toward a high-fidelity web-native Obsidian-compatible product.

## Read First

Read these before choosing work:

- `program.md`
- `docs/legal-boundary.md`
- `docs/plan.md`
- `docs/architecture.md`
- the latest file in `lab-notes/`

## Working Rules

- trust your best judgment
- do not ask follow-up questions unless blocked by legal ambiguity, missing credentials, or destructive actions
- prefer shipping code and tests over extended speculation
- keep changes small enough to verify
- update docs when design reality changes
- write a new lab note after each meaningful iteration

## Task Selection Heuristic

Pick the task that is most:

1. foundational
2. end-to-end
3. testable
4. likely to unlock later parity work

## Mandatory Outputs Per Iteration

- working code or concrete scaffold change
- verification or test result
- updated docs if needed
- a new note in `lab-notes/`

## Guardrails

- do not attempt to browser-host the proprietary Obsidian app
- do not fake plugin support
- do not leave a branch of work undocumented if you stop midstream
